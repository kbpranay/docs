# Flows and actions (Legacy)

<Callout type="warning">
  **Important — this feature is deprecated.** This article describes how **intents** trigger flows and bot actions through the **Response → Action** panel. That is the legacy path and is being phased out by the end of 2026.

  **The current, recommended path is agent tool calling.** Both **Standard (cascaded)** agents and **Realtime Voice-to-Voice** agents trigger a flow by calling it as a tool — no intent required. See [Configuring your agent → Tools](/bot-settings/agent-configuration#tools), [AI and agent nodes](/integrations/ai-agents), and [Managing flows in AIDA → How flows get triggered](/integrations/flows-basics#how-flows-get-triggered).
</Callout>

**Actions** let a legacy intent trigger something beyond a text reply — a Blitzico flow, an AIDA Flow, a knowledge lookup, or a predefined bot behavior like handing off to a live agent. You configure actions per intent from the **Response → Action** section.

## Action types

| Action | What it does |
|---|---|
| **DocBrain** | Fetch an answer from indexed documents. See [DocBrain](/intents-entities/docbrain). |
| **Blitzico** | Run a Blitzico low-code flow. Requires a valid route ID. |
| **Flows** | Run an AIDA Flow. Uses the same route ID pattern as Blitzico. |
| **Others** | Standard bot actions like `speakToAgent` or `lead_contact_details`. |

## Configuring a Blitzico or AIDA Flow action

Both Blitzico and AIDA Flows use a **route ID** to link the intent to the flow.

To create a new flow from inside an intent:

1. Open the intent you want to connect.
2. Go to the **Response** tab and click **Action**.
3. Select **Blitzico** or **Flows** from the dropdown.
4. Choose the target instance.
5. Click **Create new flow**. A new route ID is generated.
6. Click **Redirect to Blitz** (or **Redirect to Flow**) to open the flow editor in the target product.

## Using an existing flow

To reuse a flow that already exists:

1. Copy the route ID from the existing intent or flow.
2. In your intent's **Response → Action**, paste the route ID into the **Route ID** field.
3. Click **Verify**.

A valid ID turns the button label into **Redirect to Blitz** or **Redirect to Flow**, depending on the action type.

<Callout type="warning">
  **Important:** The route ID must belong to the **same instance** as the destination. Pasting an ID from a different instance shows the error *"Not a valid blitzico action"*.
</Callout>

## Configuring a standard bot action

The **Others** action category covers built-in bot behaviors.

1. Type the action name into the field.
2. Press **Enter**.
3. Click **Save**.

Common examples:

- **`speakToAgent`** — hand the conversation off to a human agent. This requires agent routing to be configured.
- **`lead_contact_details`** — capture and display the user's contact information.

Each standard action has a specific effect defined at the product level. Check with your AIDA admin for the full list of available actions in your deployment.

## Removing an action

To remove an action from an intent:

1. Open the intent.
2. Go to the **Response** tab.
3. Click **Remove action**.
4. Confirm in the dialog.

## Applying the same action to all channels

Use the **Apply to all** button to propagate the current action configuration to every response type — Generic, WhatsApp, Voice, URL, and Custom.

<Callout type="warning">
  **Warning:** Applying an action to all channels **overrides** any action already set on those channels. Use this only when you want identical behavior everywhere.
</Callout>

A blue icon appears in the header tabs (Triggers, Parameters, Response) when saved content exists, giving you a quick visual indicator of what's been configured.
