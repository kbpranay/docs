# Live Data

Live Data shows conversations as they happen. It's the fastest way to spot issues with a freshly deployed change, or to monitor a high-profile launch in real time.

## What's included

Live Data surfaces every conversation currently active across your integrations — web chatbot, voice calls, WhatsApp, and any connected channels.

For each session you can see:

- Source and channel
- Start time
- User's current message
- Bot's current response
- Whether the user has requested an agent

## Access by role

The Live Data page is accessible to:

- Team Owner
- Manager
- AIDA Admin
- AIDA User
- AIDA Dashboard

The AIDA Dashboard role has read-only access, useful for support operations or customer success teams who need visibility without editing rights.

## Using Live Data during rollouts

A good pattern after shipping a change:

1. Open Live Data in one browser tab.
2. Filter to the channel where your change is live (e.g., Webpage).
3. Watch for unexpected patterns — sudden fallback spikes, longer-than-usual conversations, more agent handoffs.
4. If something looks wrong, check the [Training page](/intents-entities/train-conversations) for the specific sessions.

For historical or aggregate analysis, use [Reports & Insights](/analytics/reports-insights) instead.
