# Working in the flow workspace

The flow **workspace** is the canvas where you build your automation. You drag nodes in from the left-hand palette, connect them, then save and deploy. This article walks through every part of the workspace UI and the common tasks you perform inside it.

## Before you begin

- You should have already created a flow. If you haven't, see [Managing flows in AIDA](/integrations/flows-basics).
- You need permission to edit flows in the current workspace. Team Owners and Admins always have access; other roles depend on your team's settings.

## The node palette

The left-hand sidebar groups every node type into six categories. Click a category icon to expand it, then drag a node onto the canvas.

| Category | Nodes | Reference |
|---|---|---|
| **Start** | Start, Trigger via API | [Start, logic, and dev nodes → Start](/integrations/logic-dev#starting-a-flow) |
| **AI tools** <sup>α</sup> | LLM, RAG, Summariser, Classifier, LLM Entity Extraction, Agentic AI | [AI and agent nodes](/integrations/ai-agents) |
| **Conv** (Conversation) | Ask User, Gallery Card, Form Card, Picture Card, Speak to an Agent, Suggestion, Video Card, LLM Classifier | [Conversation nodes](/integrations/conversation-nodes) |
| **Dev** | Comment, REST API, Function, Delay, API response, Debug | [Start, logic, and dev nodes → Dev](/integrations/logic-dev#using-dev-nodes) |
| **Logic** | If/Else, Change, Template | [Start, logic, and dev nodes → Logic](/integrations/logic-dev#branching-and-transforming-data) |
| **Link** | Flow In, Flow Out | [Start, logic, and dev nodes → Link](/integrations/logic-dev#linking-flows-together) |

<Callout type="info">
  **Note:** The **α** (alpha) badge on the AI tools category means the whole category is still evolving. Each individual AI node — LLM, RAG, Summariser, Classifier, LLM entity extraction, Agentic AI — also carries a **β** (beta) badge to indicate the node itself is still being stabilized.
</Callout>

## The workspace header

The icons across the top of the workspace, left to right:

| Icon | Name | What it does |
|---|---|---|
| 🔍 | **Search** | Find a node, variable, or label inside the current flow. |
| ⚙️ | **Flow Environment Variables** | Open the **Flow Env Variable** dialog — see [Working with flow environment variables](#working-with-flow-environment-variables). |
| ⬇ | **Export** | Download the flow as JSON, as a Blitzico file, or download just the nodes you've selected. |
| \|×\| | **Variables panel** | Open a side panel listing every variable referenced in the current flow. |
| ⬆ | **Import** | Open the **Import Flow** dialog — see [Importing a flow](#importing-a-flow). |
| ⛶ | **Fullscreen** | Expand the workspace to fill the browser window. Click again to exit. |
| **Save Flow** | Save | Persist your changes as a draft. The flow is not live yet. |
| **Deploy ▾** | Deploy | Push your changes to the live environment. The dropdown arrow lets you deploy to a different instance. |

The bottom-left corner has zoom (+ / −), fullscreen, **lock workspace**, and **mini-map** controls.

## Adding and connecting nodes

1. Drag a node from the left-hand palette onto the canvas.
2. Click the small connection-point circle on the source node.
3. Drag a wire from the source node to the next node's input.

Repeat for each node you want to add.

## Saving a flow

Click **Save Flow** in the header. Saving stores your work as a draft.

<Callout type="info">
  **Note:** Saving is not the same as deploying. You can save as many draft versions of a flow as you want — none of them affect the live bot until you deploy.
</Callout>

## Deploying a flow

To push your changes to the live environment:

1. Click **Save Flow** to save the current state.
2. Click **Deploy** in the top-right corner.

Your changes are now live on the bot.

## Deploying to another instance

If your team has multiple instances (for example, staging and production), you can deploy the same flow to each.

1. Save the flow in your current instance.
2. Click the dropdown arrow next to the **Deploy** button.
3. Select **Choose Instance** and confirm.
4. Pick the target instance.
5. Click **Deploy**.

## Exporting a flow

From the workspace header, click the **Export** (⬇) icon and choose one of the following options.

| Option | What it does |
|---|---|
| **Export as Blitzico File** | Download the flow as a Blitzico-compatible file. |
| **Export Flow** | Download or copy the full flow as JSON. |
| **Export Selected Nodes** | Download or copy only the nodes you've selected. |

<Callout>
  **Tip:** Use **Export Selected Nodes** to share reusable sub-flows between teammates without sending the whole flow.
</Callout>

## Importing a flow

To bring a flow in from a file or from a JSON snippet:

1. Click the **Import** (⬆) icon in the workspace header. The **Import Flow** dialog opens.
2. Either:
   - Paste a flow JSON payload into the **Paste flow json** textarea, or
   - Click **Select File to Import** and choose a `.json` file (or a Blitzico file) exported earlier.
3. Click **Import**.

The nodes drop onto the canvas and you can rearrange or edit them as needed.

## Working with flow environment variables

Flow environment variables are values that stay with the flow — API keys, configuration defaults, and anything else you want every node in the flow to have access to without hardcoding it.

To open the environment variables dialog:

1. Click the ⚙ **Flow Environment Variables** icon in the workspace header. The **Flow Env Variable** dialog opens.
2. Click **+ Add**.
3. Enter a **Name**, **Value**, and **Type**.
4. Click **Save**.

<Callout type="info">
  **Note:** Variables defined here are specific to the current flow. They aren't shared with other flows. If you need a value across multiple flows, define it in each one — or handle it at the team level.
</Callout>

For details on referencing these variables from nodes, see [Data and tool integrations → Environment variables](/integrations/data-tools#using-environment-variables).

## Viewing flow logs

Open **Flow History** from the top header to see every deployment and change event. Click **Export Logs** to download the history as a CSV file.

## Annotating flows with comments

Drag a **Comment** node from the **Dev** palette onto the canvas, then double-click the node to edit the text. Comment nodes don't execute — they exist only to document the flow for your future self and your teammates.

## Navigating the canvas

| Feature | How to use it |
|---|---|
| **Fullscreen** | Click the fullscreen icon in the workspace header. Click again to exit. |
| **Mini-map** | Click the mini-map icon in the bottom-left to show a zoomed-out overview. Drag inside the mini-map to pan the canvas. |
| **Lock workspace** | Click the lock icon in the bottom-left to prevent accidental edits. |
| **Zoom** | Scroll with your mouse or trackpad, or use the zoom buttons in the bottom-left. Click the reset button to return to the default zoom. |
| **Multi-select** | Hold **Shift** and click multiple nodes, or click and drag an empty area to box-select. Multi-selected nodes can be moved, copied, or deleted in bulk. |

## Debugging a flow in Picture-in-Picture

To open the debug window:

1. Select the flow you want to debug.
2. Click the **Debug** icon in the bottom-right of the workspace.
3. Click the **Extension** icon for extra debug features.

<Callout type="warning">
  **Important:** Picture-in-Picture debugging is supported in **Chrome** only. Firefox and Safari don't support the underlying browser feature.
</Callout>

## Sending product feedback

A **🐛 feedback** button is anchored to the bottom-right corner of the workspace. Click it to report a bug or send feedback about the flow builder directly to the AIDA team — useful for quick reports without leaving the workspace.

## Keyboard shortcuts

The workspace uses your platform's standard modifier key:

| Platform | Primary modifier |
|---|---|
| macOS | **⌘ (Command)** |
| Windows / Linux | **Ctrl** |

Combine the modifier with standard shortcuts (copy, paste, undo, redo, select all) to edit flows quickly.
