# Workspace

The Flows workspace is where you build the actual automation — drag nodes from the palette, wire them together, and deploy.

## Node palette (left sidebar)

The left-hand sidebar groups all node types into six categories. Click a category icon to expand it and drag a node onto the canvas.

| Category | Nodes | Reference |
|---|---|---|
| **Start** | Start, Trigger via API | [Start, Logic & Dev Nodes → Start](/integrations/logic-dev#start-start-node) |
| **AI tools** <sup>α</sup> | LLM, RAG, Summariser, Classifier, LLM Entity Extraction, Agentic AI | [AI & Agents](/integrations/ai-agents) |
| **Conv** (Conversation) | Ask User, Gallery Card, Form Card, Picture Card, Video Card, Suggestion, Speak to an Agent, LLM Classifier | [Conversation Nodes](/integrations/conversation-nodes) |
| **Dev** | Comment, REST API, Function, Delay, API response, Debug | [Start, Logic & Dev Nodes → Dev](/integrations/logic-dev#dev-rest-api) |
| **Logic** | If/Else, Change, Template | [Start, Logic & Dev Nodes → Logic](/integrations/logic-dev#logic-ifelse) |
| **Link** | Flow In, Flow Out | [Start, Logic & Dev Nodes → Link](/integrations/logic-dev#link-flow-in--flow-out) |

<Callout type="info">
  The **α** (alpha) badge on **AI tools** indicates the category is in active development — capabilities expand frequently and some behaviors are still being stabilized.
</Callout>

## Workspace header (top bar)

The icons across the top of the workspace, left-to-right:

| Icon | Name | What it does |
|---|---|---|
| 🔍 | **Search** | Find a node, variable, or label inside the current flow |
| ⚙️ | **Flow Environment Variables** | Opens the **Flow Env Variable** dialog — see [Flow environment variables](#flow-environment-variables) below |
| ⬇ | **Export** | Download the flow as JSON, a Blitzico file, or just the selected nodes |
| \|×\| | **Variables panel** | Open the side panel listing every variable referenced in this flow (flow env vars, global context, message properties) |
| ⬆ | **Import** | Opens the **Import Flow** dialog — see [Importing a flow](#importing-a-flow) below |
| ⛶ | **Fullscreen** | Expand the workspace to fill the browser window |
| **Save Flow** | Save | Persist changes as a draft (not live) |
| **Deploy ▾** | Deploy | Push to the live environment; the dropdown lets you deploy to another instance |

The bottom-left corner has zoom (+ / −), fullscreen, **lock workspace**, and **mini-map** controls.

## Adding and connecting nodes

- **Add:** drag a node from the left-hand sidebar palette and drop it onto the workspace.
- **Connect:** click the small connection-point circle on the source node and draw a wire to the next node.

## Deploying a flow

<Steps>

### Save your flow

Click **Save Flow** at the top of the workspace. Saving keeps your work, but the flow isn't live yet.

### Click Deploy

The **Deploy** button is in the top-right corner. Clicking it applies your changes to the live environment.

</Steps>

<Callout>
  **Save ≠ Deploy.** You can save draft versions of a flow indefinitely. Only a deploy pushes them to the live bot.
</Callout>

### Deploying to another instance

If your team has multiple instances (e.g., staging and production), you can deploy the same flow across them:

<Steps>

### Save the flow

Save it in your current instance.

### Click the Deploy dropdown arrow

Next to the **Deploy** button.

### Choose Instance

Click **Choose Instance → Confirm**, then select the target instance.

### Deploy

Click **Deploy** to push the flow to that instance.

</Steps>

## Export and import

**Export** from the top header:

| Option | What it does |
|---|---|
| **Export as Blitzico File** | Download the flow as a Blitzico-compatible file |
| **Export Flow** | Download or copy the full flow as JSON |
| **Export Selected Nodes** | Download or copy just the nodes you selected |

### Importing a flow

Click the **Import** (⬆) icon in the header to open the **Import Flow** dialog.

- **Paste flow json** — paste a JSON payload straight into the textarea
- **Select File to Import** — pick a `.json` file exported earlier (or a Blitzico file)

Click **Import** to drop the nodes onto the canvas, or **Cancel** to dismiss.

## Flow environment variables

Click the **⚙ Flow Environment Variables** icon in the header to open the **Flow Env Variable** dialog. These values live on the flow itself — they're accessible to every node in this flow but not shared across flows.

<Steps>

### Open the dialog

Click the ⚙ icon in the workspace header.

### Add a variable

Click **+ Add** and enter a Name, Value, and Type.

### Save

Click **Save** to persist. You'll see the callout *"Variables defined are specific to this flow"* — intentional, to remind you these aren't team-wide.

</Steps>

Reference them from nodes as described in [Data & Tool Integrations → Environment variables](/integrations/data-tools#environment-variables).

## Flow logs

Open **Flow History** from the top header. Click **Export Logs** to download a CSV of every deployment and change event.

## Comments

Drop a **Comment** node from the Dev section to annotate your flow. Double-click the node to rename or edit the comment text. Comments are non-executing — they exist purely for documentation.

## Navigating the canvas

| Feature | How |
|---|---|
| **Fullscreen** | Click the fullscreen icon in the workspace header. Click again to exit. |
| **Mini map** | Icon in the bottom-left — shows a zoomed-out overview. Drag inside to pan. |
| **Lock workspace** | Lock icon at the bottom-left prevents accidental changes. |
| **Zoom** | Scroll with mouse/trackpad, or use the zoom buttons bottom-left. A reset button restores default. |
| **Multi-select** | Hold **Shift** + click, or hold and drag to box-select. Enables bulk move/delete/copy. |

## Debugging in Picture-in-Picture

Select a flow, click the **Debug** icon (bottom-right) to open the debug window. Click the **Extension** icon for additional debug features.

<Callout type="warning">
  Picture-in-Picture debugging is **Chrome browser only**. Firefox and Safari don't support the feature.
</Callout>

## Hotkeys

| Modifier | macOS | Windows / Linux |
|---|---|---|
| Primary modifier | **⌘ (Command)** | **Ctrl** |

Hotkeys follow standard patterns — combine the modifier above with common shortcuts (copy, paste, undo, select all) to speed up flow editing.
