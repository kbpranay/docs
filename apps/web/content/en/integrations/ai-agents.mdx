# AI and agent nodes

Flows include seven AI nodes that cover everything from a single LLM call to an autonomous agent that plans and executes multi-step actions. This article explains each node, when to use it, and how to configure it.

<Callout type="info">
  **Note:** In the workspace palette, these nodes live under **AI tools** and are marked with an **α** (alpha) badge. The category is under active development — new capabilities are added regularly and some behaviors are still being stabilized.
</Callout>

## Before you begin

- You should know which LLM your team has access to. The models available in the Model dropdown depend on your workspace and mode (Standard or Realtime Voice).
- For tools that read from a knowledge base (RAG, KB tool), make sure you have a DocBrain index ready. See [DocBrain](/intents-entities/docbrain) for how to create one.

## LLM node

Use the **LLM** node to call an LLM service with a system prompt and branch on whether the model responds.

**System prompt** controls tone, style, context, and safety guidelines.

**Advanced settings:**

- Model
- Temperature (0–1; higher values produce more creative output)
- Max Output Tokens
- Request Timeout
- Retries
- PII handling
- Chat Context

**Outputs:**

- **Success** — the LLM returned a response.
- **Otherwise** — the LLM didn't return a response.
- **Catch Error** — something went wrong during execution.

## RAG node

Use the **RAG** node to combine real-time knowledge retrieval with LLM generation. RAG grounds the response in your indexed documents (DocBrain, EnterpriseBot's patented retrieval technology) so the model hallucinates less.

**Modes:**

- **RAGForge** — reuse the bot's existing RAG config. You can still override specific settings.
- **Custom** — pick the LLM and the configuration yourself.

**General configuration:**

- `allow_links`, `allow_smalltalk`, `allow_comparison`
- `add_current_date` (with timezone)
- `use_bullet_points`, `use_titles`
- **Basic Prompt** / **Fallback Event**
- **Only LLM Reference URLs** / **Reset Context**

**UI elements:**

- Gallery Card, Picture Card
- Show Reference URLs (with Topk URLs and Reference Title)

**Hallucination check:**

- Toggle + fallback event
- Optional **LLM Validator** (a separate model with its own system prompt)

**Advanced settings:**

- **Method** — `Gpt_mix_train_faq+para`, `Gpt_mix_train_faq`, or `Mix_train_faq`
- **Type** — Semantic or Keyword
- Max Output Tokens, Request Timeout, Topk, Show score
- Intent Context, Context Length
- Slack Notification + channel ID

**Guardrails:**

| Input rules | Output rules |
|---|---|
| Limit user message length | Rule / Prompt leak detection |
| Prompt injection detection | Black list |
| Black list |  |
| Abbreviation expansion |  |

**PII anonymiser:**

- Anonymise PII toggle
- Choose PII Type (Query or All)
- Default masks: IBAN, phone, email, person names
- Custom values via Regex or Keyword

## Summariser node

Use the **Summariser** node to condense large inputs into a short summary through the LLM Summariser service. It returns **Success** or **Otherwise**.

The configuration mirrors the LLM node, plus a **Context limit** field that appears when Chat Context is on.

The system prompt controls style, length, format, and accuracy.

## Classifier node

Use the **Classifier** node to categorise inputs using NLP for context, semantics, and tone.

**Configure:**

- **Topic** — the main category.
- **Description** — optional context for the topic.
- **Variation** — alternative keywords or phrases for the topic.

**Advanced settings:** the same fields as the LLM node (Model, System Prompt, Temperature, Max Output Tokens, Request Timeout, Retries, PII handling, Chat Context, Context limit).

## LLM Entity Extraction node

Use the **LLM Entity Extraction** node to pull structured entities — names, dates, locations, amounts — out of unstructured text using the LLM NER service.

**Configure:**

- **Entity Name** — what to extract.
- **Description** — context to guide the extraction.

Advanced settings match the LLM node.

## Agentic AI node

An **agent** is a digital teammate that can think, plan, and act autonomously — not just answer a single question.

An agent has four capabilities:

- **Memory** — remember past interactions.
- **Reasoning** — plan multi-step actions.
- **Tool use** — invoke external tools to gather information or take actions.
- **Execution** — carry plans out end to end.

**Tools your agent can use:**

| Tool | What it does |
|---|---|
| **API** | Call external services — CRM, ticketing, payments, translation. |
| **KB** | Use a DocBrain or Knowledge Base as a reference source (RAG-style). |
| **Table** | Query structured CSV data with SQL-like queries. |
| **File** | Read PDF content to generate responses. |
| **SQL** | Run read-only SQL queries on connected databases. |
| **Webpage** | Fetch and read live webpage content — offers, discounts, status pages. |

For the full setup of each tool type, see [Data and tool integrations](/integrations/data-tools).

## Creating an agent

To create a new agent from the **Agents** page:

1. Open the **Agents** page and click **Create**.
2. Fill in the following fields:
   - **Name**
   - **Description** — natural-language instructions for the agent.
   - **Enhance Prompt** — click to auto-expand the description.
   - **Expected Output** — the format you want back.
   - **Tools** — the tools the agent can use.
3. Review the **Advanced Setup** options: Model, Add history to messages, Number of history responses, Add datetime to instructions, Timezone, Tool call limit, Markdown.
4. Click **Create**.

## Using an agent in a flow

To place an existing agent or create a new one from inside a flow:

1. Open the flow in the workspace.
2. Drag an **Agentic AI** node from the **AI tools** palette onto the canvas.
3. Either:
   - Click **Create Agent** to define a new agent inline (same fields as the Agents page), or
   - Pick an existing agent from the **Select Agent** dropdown. The node's configuration is auto-populated.

## Triggering a flow from an agent (tool call)

Agents can also launch flows — register a flow as a tool on the agent and the agent calls it when the conversation needs it.

1. Open **Agent Configuration** for your agent.
2. Go to the **Tools** tab.
3. Click **+** to open the **Add Tools** dialog.
4. Enter a name and a clear **Description** of what the flow does and when the agent should call it.
5. Finish the rest of the tool configuration and click **Save**.

This works the same way in both agent modes:

| Agent mode | How the flow is triggered |
|---|---|
| **Standard (cascaded)** | The agent plans the response and invokes the flow as one of its tool calls. |
| **Realtime Voice-to-Voice** | Low-latency voice agents (Gemini Live, GPT-realtime) invoke the flow mid-conversation without leaving the voice session. |

<Callout type="warning">
  **Important:** The legacy way to trigger a flow was to route an intent's **Response → Action** to the flow's route ID. This still works for intent-based bots but is being phased out by the end of 2026. New bots should use agent tool calls instead. See [Managing flows in AIDA → How flows get triggered](/integrations/flows-basics#how-flows-get-triggered).
</Callout>

## Inspecting an agent's response pipeline

When an agent runs, you can inspect each step of the pipeline to understand why the agent made the decisions it did.

| Section | What you see |
|---|---|
| **Request Details** | Intent *(legacy field — populated only for intent-triggered runs)*, Action, LLM used, Message ID, Response Time, Agent Prompt. |
| **Tools Added** | The list of tools configured on the agent, including each tool's parameters. |
| **Response Section (per tool call)** | System Prompt, Context, Result (the arguments passed to the tool), and the LLM's response. |

This makes debugging much easier — you can see exactly what the agent looked at, what it called, and what it returned at each step.
