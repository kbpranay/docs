# Conversation nodes

Conversation nodes — shown as **Conv** in the palette — drive the visible back-and-forth of the conversation. You use them to prompt users for input, render rich cards, and branch based on how the user replies.

## Before you begin

- You should already have a flow open in the workspace. See [Managing flows in AIDA](/integrations/flows-basics).
- Rich-media cards (Gallery, Picture, Video) need public URLs for their images and videos. Make sure you have those ready.

## Ask User node

Use the **Ask User** node to prompt the user for free-text input and store the reply in a variable.

**Fields:**

- **Message** — the prompt the user sees.
- **Conditions** — validation rules applied to the reply.
- **Fallback Message** — shown when validation fails.
- **Variable** — where the reply is stored.
- **Suggestions** — optional quick-reply chips.
- **Show text area** — toggle between a full textarea and a single-line input.

## Gallery Card node

Use the **Gallery Card** node to display a horizontally swipeable carousel of cards — useful for product catalogs, option lists, or article suggestions.

**Per card:**

- Card Image URL
- Float Image URL
- Title
- Subtitle
- Description
- Add Button (multiple buttons per card are supported)

**Around the carousel:**

- Message Before Card
- Message After Card
- Suggestions
- Fallback Message
- Variable
- Show text area

## Form Card node

Use the **Form Card** node to show a structured form inline in the chat. The user submits multiple fields at once, and each field can be validated.

**Field types:**

| Field | Use for |
|---|---|
| Header | Visual grouping |
| TextField | Single-line text |
| Email | Email addresses (validated automatically) |
| Text Area | Multi-line text |
| Date Picker | Dates |
| Time Picker | Times |
| Numeric | Numbers |
| Dropdown select | Single choice from a list |
| Checkbox group | Multiple choices |
| Radio button | Single choice, with every option visible |

Each **Action Button** supports a **Label**, **Validation text**, and **Help** text. The surrounding configuration includes the Message before card, Message after card, Mandatory message, and the Show text area toggle.

## Picture Card node

Use the **Picture Card** node to display one or more images with optional surrounding text.

**Fields:**

- Message Before Card
- Picture (Image URL)
- Add card — append more images
- Message After Card
- Suggestions
- Fallback Message
- Variable
- Show text area

## Video Card node

Use the **Video Card** node to embed a video — perfect for walkthroughs, demos, or onboarding content.

**Fields:**

- Message Before Card
- Video cards (add a Card Video URL for each card)
- Message After Card
- Suggestions
- Fallback Message
- Variable
- Show text area

## Suggestion node

Use the **Suggestion** node as a lightweight way to present quick-reply chips — good for short menus or follow-up prompts.

**Fields:**

- Message
- Suggestions
- Fallback Message
- Variable
- Show text area

## Speak to an Agent node

Use the **Speak to an Agent** node to hand the conversation off to a live agent using a suggestion chip. The text shown to the user adapts based on the live agent's availability status.

**Fields:**

- Suggestions
- Fallback Message
- Show text area

## LLM Classifier node

Use the **LLM Classifier** node to ask the user a question, classify their response into one of several topics, and branch the flow accordingly.

**Fields:**

- **Question** — what to ask the user.
- **Description** — optional context for the classifier.
- **Topic(s)** — each topic becomes an output branch.
- **Topic Description** — helps the LLM pick the right topic.
- **Fallback Message** — shown when no topic matches.
- **Enable Fallback flow** — adds an **LLM Fallback Flow** output with a **Maximum Retry Limit** (1–5; default is 3).
- **Variable** — stores the topic that was chosen.
- **Topic(s) variables** — per-topic variable storage.
- **Show text area**

<Callout type="warning">
  **Important:** If you don't enable the fallback flow, the conversation terminates after **4 retries** with a default AIDA message. We recommend enabling the fallback flow on any classifier where you want graceful recovery from misclassification.
</Callout>
