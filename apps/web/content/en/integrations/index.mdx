# Integrations and flows overview

**Flows** is AIDA's visual automation builder. You use it to connect your bot to external systems — APIs, databases, knowledge bases, AI services — without writing a full application. You build a flow by dragging nodes onto a canvas and wiring them together, then expose that flow to your agent as a tool the agent can call during a conversation.

This article introduces flows and points you to the right article for the task you want to do.

## What you can do with flows

Use flows to:

- **Call external APIs** — CRMs, ticketing systems, payments, translation services.
- **Query data** — run SQL against a read-only database, query a CSV table, or read from a DocBrain knowledge base.
- **Work with AI** — combine LLM, RAG, classifier, entity-extraction, and full Agentic AI nodes in a single flow.
- **Build multi-step logic** — branch with If/Else, transform data with Change and Template, pace execution with Delay.
- **Hand off to a human** — route the conversation to a live agent when the bot can't resolve the request.

### Why flows?

- **Visual, not code-heavy** — design complex logic by wiring nodes instead of writing code.
- **Everything in one place** — no switching between a bot builder and a separate integration tool.
- **Versioned and auditable** — every change is logged with who did it and when; you can revert at any time.
- **Lockable** — lock a flow to prevent accidental edits while it's running in production.
- **Preview at design time** — card nodes show inline previews so you can validate layouts before shipping.

## How flows fit with the rest of AIDA

At runtime, a flow is one of the tools an agent can use:

- **Agents** read the conversation and decide what's needed. They pick the right tool — a flow, a knowledge base, an API, and so on — and call it.
- **Flows** execute the integration logic. They call external systems, fetch data, and shape the response.
- **Channels** deliver the result back to the user on the web widget, voice, WhatsApp, or whichever channel the conversation started on.

Both agent modes can launch a flow the same way:

- **Standard (cascaded) agents** — text with STT/TTS. The LLM plans and calls the flow like any other tool.
- **Realtime Voice-to-Voice agents** — low-latency voice built on models like Gemini Live or GPT-realtime. The agent calls the flow mid-conversation without leaving the voice session.

You register a flow as an agent tool from **Agent Configuration → Tools**. See [Configuring your agent](/bot-settings/agent-configuration#tools) for the full walkthrough.

<Callout type="warning">
  **Important:** Intents used to trigger flows through a **Response → Action** route ID. This still works for bots that were built before AIDA moved to agents, but it's deprecated and being phased out by the end of 2026. For any new work, use agent tool calls instead. See [Intents and entities (Legacy)](/intents-entities).
</Callout>

## How to open the Flows page

You can open the Flows listing page two ways:

1. From the **Bot Selection** page, click **Flows**.
2. From the top navigation, open the **Mega Menu** and click **Flows**.

If you're using the modern agent-based path, you normally won't open a specific flow by its route ID — the agent launches the flow through its tool configuration. For legacy intent-based bots, you can still jump to a specific flow by pasting its route ID into **Intent → Response → Action**.
