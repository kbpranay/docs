# Managing flows in AIDA

A **flow** is a visual automation you build inside AIDA to orchestrate API calls, AI agents, database queries, and rich conversational responses. You manage all of your flows from the **Flows** listing page. From there you can create new flows, rename or delete existing ones, lock flows that are running in production, and search the list.

This article walks you through each of those tasks and explains how a flow actually gets triggered at runtime.

## Before you begin

Before you work with flows, make sure:

- You have access to the AIDA workspace that contains the bot or agent you're building.
- You know which **instance** (for example, staging or production) the flow should deploy to.
- If you're building a modern agent-based bot, you've already created the agent (see [Creating your first agent](/getting-started/first-bot)). The flow you create here will become one of the tools the agent can call.

## How flows get triggered

Before you create a flow, it helps to understand how AIDA decides to run one. A flow can be launched in three ways, but only the first two are recommended for new bots.

| Trigger | When it applies | Status |
|---|---|---|
| An **agent in Standard (cascaded) mode** calls the flow as a tool | Text-based or STT/TTS agents that plan a response and invoke tools step by step | **Recommended** |
| An **agent in Realtime Voice-to-Voice mode** calls the flow as a tool | Low-latency voice agents built on models like Gemini Live or GPT-realtime | **Recommended** |
| A **legacy intent** routes to the flow through **Response → Action** and a route ID | Only for bots that were built before AIDA moved to agents | **Deprecated** — being phased out by the end of 2026 |

<Callout type="info">
  **Note:** Agents call flows exactly like any other tool. The agent reads the **Description** you gave the flow when you registered it as a tool, decides from the conversation whether the flow is needed, calls it, and uses the result to continue the conversation. Both Standard and Realtime Voice-to-Voice agents work this way.
</Callout>

<Callout type="warning">
  **Important:** The "trigger a flow from inside an intent" pattern is kept only for teams maintaining existing intent-based bots. If you're building a new bot, create an [agent](/getting-started/first-bot) and register the flow as one of its tools instead.
</Callout>

## Creating a flow

Every flow starts as an entry on the Flows listing page. To create a new flow:

1. Open the **Flows** page from the Bot Selection page or from the Mega Menu in the top navigation.
2. Click **+ Create**. The **Create Flow** dialog opens.
3. Enter a descriptive name for your flow.
4. Choose the **instance** you want the flow to deploy to.
5. Click **Save**.

The new flow appears in the list and the workspace opens so you can start adding nodes.

<Callout>
  **Tip:** Name flows after the business outcome they deliver — for example, `CheckOrderStatus` or `RescheduleAppointment` — so that your teammates (and your agents) can pick the right one at a glance.
</Callout>

## Registering a flow as an agent tool (recommended)

To let an agent launch the flow during a conversation:

1. Open **Agent Configuration** for the agent that should use this flow.
2. Go to the **Tools** tab.
3. Click the **+** button next to the search field to open the **Add Tools** dialog.
4. Fill in the tool name and a clear **Description**. The description is what the agent reads to decide when to call your flow, so be specific about what the flow does and when it should run.
5. Finish the rest of the tool configuration (method, URL, parameters) and click **Save**.

The tool immediately appears in the agent's available tool list and the agent can call it on the next conversation turn. The configuration is identical for **Standard (cascaded)** and **Realtime Voice-to-Voice** agents.

For the full walkthrough, see [Configuring your agent → Tools](/bot-settings/agent-configuration#tools).

## Triggering a flow from an intent (legacy)

<Callout type="warning">
  **Important:** This section describes the legacy intent-based path. It's kept only for teams maintaining intent-based bots and is being phased out by the end of 2026. Use the agent path above for any new work.
</Callout>

If you're maintaining a legacy bot, you can still link a flow to an intent:

1. Open the intent you want to connect to a flow.
2. Go to the **Response** tab and click **Action**.
3. Select **Flow** from the action-type dropdown.
4. Either click **Create new flow** to generate a new route ID, or paste an existing route ID and click **Verify**.
5. Save the intent.

## Renaming a flow

To change a flow's name:

1. On the Flows page, find the flow in the list.
2. Click the meatball menu (the three dots) next to the flow.
3. Choose **Edit Name**.
4. Enter the new name and confirm.

The change is reflected immediately in the list.

## Deleting a flow

<Callout type="warning">
  **Warning:** Deleting a flow is permanent. Any agents that have this flow registered as a tool — and any legacy intents or parent flows that reference its route ID — will stop working the moment you delete it.
</Callout>

To delete a flow:

1. Open the meatball menu next to the flow in the list.
2. Click **Delete Flow**.
3. Confirm the deletion in the popup.

Before you delete, we recommend:

- Checking which agents have this flow registered as a tool and removing the tool from each of them first.
- For legacy bots, searching your intents for the flow's route ID so you can update or remove those references.

## Locking a flow

Locking a flow puts it in view-only mode so that nobody can edit it until it's unlocked. This is useful for production flows that should only change under review.

To lock a flow:

1. Open the meatball menu next to the flow in the list.
2. Click **Lock Flow**.
3. Confirm the action.

<Callout>
  **Note:** When a flow is locked by one user, the meatball menu is hidden for everyone else so they can't accidentally take actions on it. The person who locked the flow — or any admin — can unlock it through the same menu.
</Callout>

## Searching for a flow

The **Search Flow** field at the top of the Flows page filters the list as you type. Matches highlight instantly; you don't need to press **Enter**.

## Viewing a flow's change history

Every flow keeps a comprehensive change log. The log shows:

- Who made each change
- When the change happened
- What was changed

To open the history panel, click the history icon in the top header of the workspace. From there you can also revert the flow to any previous version.
