# Flow Management

Every flow starts as an entry on the Flows listing page. From there you can create, rename, delete, lock, and search for flows.

## How flows get triggered

Flows can be launched in three ways. **Agent tool calls are the current, recommended approach**; intent-action triggers are kept only for legacy, intent-based bots.

| Trigger | Mode | Status |
|---|---|---|
| **Agent (Standard / cascaded)** calls the flow as a tool | Text + STT/TTS agents that plan, then invoke tools step-by-step | **Recommended** |
| **Agent (Realtime Voice-to-Voice)** calls the flow as a tool | Low-latency voice agents (Gemini Live, GPT-realtime) invoking tools mid-conversation | **Recommended** |
| **Intent → Response → Action** routes to the flow via route ID | Legacy intent-based bots only | **Deprecated** — being phased out by end of 2026 |

<Callout type="info">
  **Agents trigger flows via tool calls.** Register the flow as a tool on the agent (see [Agent Configuration → Tools](/bot-settings/agent-configuration#tools) and [AI & Agents](/integrations/ai-agents)). Both **cascaded/Standard** agents and **Realtime Voice-to-Voice** agents can invoke flows the same way — the LLM decides when to call the flow based on the tool's description and the user's request, then the flow runs and returns its result to the agent.
</Callout>

<Callout type="warning">
  **Intents are deprecated.** The "trigger a flow from inside an intent" pattern described below remains only for teams maintaining legacy bots. For new work, build an [Agent](/getting-started/first-bot) and expose the flow as a tool.
</Callout>

## Creating a flow

<Steps>

### Open the Flows page

Click **Flows** from the Bot Selection page or from the Mega Menu in the top nav.

### Click + Create

The **Create Flow** dialog opens.

### Name the flow

Enter a descriptive name and configure the instance where the flow should deploy.

### Save

Click **Save**. The new flow appears in the list and opens the workspace so you can start adding nodes.

</Steps>

### Creating a flow from an agent (recommended)

Open your agent in [Agent Configuration → Tools](/bot-settings/agent-configuration#tools), add the flow as a tool, and give it a clear **Description** so the agent knows when to call it. This works identically for **Standard (cascaded)** agents and **Realtime Voice-to-Voice** agents.

### Creating a flow from inside an intent (legacy)

<Callout type="warning">
  Applies to legacy, intent-based bots only. New bots should trigger flows via agent tool calls (above).
</Callout>

You can also create a flow from inside an intent — open **Response → Action**, select **Flow** from the dropdown, and either create a new one or paste an existing route ID.

## Renaming a flow

<Steps>

### Open the flow menu

Click the meatball menu (three dots) next to the flow in the list.

### Choose Edit Name

Update the name in the prompt and confirm.

</Steps>

## Deleting a flow

<Steps>

### Open the flow menu

Click the three dots next to the flow.

### Delete Flow

Click **Delete Flow** and confirm in the popup.

</Steps>

<Callout type="warning">
  Deleting a flow removes it permanently. Any agents that have this flow registered as a tool — and any legacy intents or parent flows that reference the flow's route ID — will stop working.
</Callout>

## Locking a flow

Locking a flow puts it in view-only mode — nobody can edit it until it's unlocked. This is useful for production flows that shouldn't change without review.

<Steps>

### Open the flow menu

Click the three dots next to the flow.

### Lock Flow

Click **Lock Flow** and confirm.

</Steps>

<Callout>
  When a flow is locked by one user, the meatball menu is **hidden for other users** — they can't accidentally take actions on a locked flow. The person who locked it (or any admin) can unlock it through the same menu.
</Callout>

## Searching flows

The **Search Flow** field at the top of the Flows page filters the list as you type. Matches highlight instantly — no need to press Enter.

## Change history

Every flow keeps a comprehensive change log that shows:

- Who made each change
- When it happened
- What was changed

You can revert any flow to a previous version through the history panel. Access it from the top header of the workspace.
