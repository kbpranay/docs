# Creating Your First Agent

AIDA has moved to an **Agentic AI** model. Instead of building chatbots from intents, triggers, and entities, you now create an **agent** with a system prompt, connected tools, and a model. AIDA walks you through it in a three-step wizard.

<Callout type="warning">
  **Intents are being deprecated.** Existing intent-based bots continue to work, but all new bots should be built as agents. Intents, triggers, and entities will be fully phased out by **end of 2026**. See [Intents & Entities (Legacy)](/intents-entities) if you're maintaining an existing bot.
</Callout>

## Step 1 — Agent Mode

Choose how your agent interacts with users.

| Mode | When to pick it |
|---|---|
| **Standard** | Text-based assistants with Speech-to-Text (STT) and Text-to-Speech (TTS) support. Great when flexibility and tool usage are priorities. Can be used as a flow node (node settings only). Guardrails still under development. |
| **Realtime Voice** | Low-latency, real-time voice interactions using natural speech. Optimized for phone calls and live conversations — input and output flow seamlessly with minimal delay. Ideal for conversational AI that mimics human-like interactions. Guardrails and PII are still under development and not yet enforced. |

<Callout>
  If your agent needs to call external tools (APIs, knowledge bases, databases), pick **Standard** — it has full tool-calling support today.
</Callout>

Click **Continue** after selecting.

## Step 2 — Company Info

Teach your agent about your business. AIDA uses this information to build a default system prompt, so the more specific you are here, the less you'll edit later.

The form asks for: **Agent Name**, **Primary Language**, **Company Name**, **Website URL**, **Welcome message**, and a short description of **what your AI assistant should do**.

<Steps>

### Name your agent

**Agent Name** — a friendly name users will see (e.g., `Alex`, `Sarah`, `Jordan`).

### Pick a primary language

**Primary Language** — the language your agent speaks by default. You can add supported languages later from the Language tab.

### Add company details

- **Company Name** — used in the agent's introduction
- **Website URL** — lets the agent reference your website in responses

### Write a welcome message

**Welcome message** — describe how your agent should greet users. One or two sentences is enough. Example: *"Hi, I'm Alex from Acme. I can help with order status, returns, and product questions."*

### Describe what the agent should do

**What should your AI assistant do?** — plain-English description of the agent's responsibilities. AIDA auto-generates a detailed system prompt from this, so be specific.

Example input:

> *Answer customer questions, book appointments, take orders, and check order status.*

</Steps>

Click **Continue** to provision the agent.

## Step 3 — Test It

AIDA spins up your agent with a default model and voice and hands you a live test console that shows the agent's key details next to an interactive preview.

You'll see:

- **Model** — the LLM powering the agent (for Realtime Voice, typically Gemini Live or a GPT-realtime variant)
- **Voice Name** — the TTS/Realtime voice (e.g., `Puck`)
- **Language** — the primary language you picked

**Try it:**

- Hit the **mic** to start talking (Realtime Voice) or type into the chat (Standard)
- Wait for the **Ready to talk** indicator to go green
- Ask your agent something it should handle based on your description

<Callout type="info">
  **Realtime Voice tips:**
  - Speak clearly and naturally — the agent responds when you finish a thought
  - Use a quiet environment for best transcription quality
</Callout>

When you're ready to fine-tune, click **Go to Agent Configuration**. See the full [Agent Configuration](/bot-settings/agent-configuration) reference for prompts, model, tools, voice, and language settings.
