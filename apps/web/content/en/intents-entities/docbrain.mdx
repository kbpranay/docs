# DocBrain

<Callout type="info">
  **DocBrain is still supported.** DocBrain is the retrieval layer for both legacy intent bots *and* new agents — new agents use DocBrain via the [KB Tool](/integrations/data-tools#kb-knowledge-base-tool). This page documents DocBrain attached to intents; for the agent-based flow, see [Data & Tool Integrations](/integrations/data-tools).
</Callout>

DocBrain lets your bot answer questions based on documents you've indexed — FAQs, manuals, policy PDFs, web pages. Instead of building every response by hand, you upload the source material and DocBrain handles retrieval and summarization.

## When to use DocBrain

Use DocBrain when:

- You have a large corpus of reference material (policies, product documentation, knowledge base articles).
- Users ask open-ended questions whose answers live in that material.
- You want to avoid hand-authoring hundreds of intents for every possible phrasing.

Use classic [Intents](/intents-entities/intents) when:

- The user's goal leads to a specific action (booking, account update, escalation).
- You need to collect structured information with [Parameters](/intents-entities/triggers-parameters#parameters).

You can mix both in the same bot — intents for transactional flows, DocBrain for knowledge lookups.

## Creating a DocBrain bot

The easiest way is during bot creation. See [Creating Your First Bot](/getting-started/first-bot#create-bot-using-docbrain).

## Connecting DocBrain to an intent

You can also attach DocBrain to a single intent as an action:

<Steps>

### Open the intent

Navigate to the Intents page and open the intent you want to power with DocBrain.

### Switch to the Response tab

Click the **Response** tab.

### Add a DocBrain action

Under Action, select **DocBrain**. You can either:
- Link an **existing DocBrain** (filtered to ones matching your bot's language), or
- **Create a new DocBrain** — this redirects to the Create DocBrain page where you can configure it and click **Start indexing**.

### Add triggers for the questions

Go back to the Triggers tab and add example questions users might ask about the indexed documents. The bot will match these triggers and fetch a DocBrain response at Go-Live.

</Steps>

## RAG configuration

Advanced DocBrain settings — prompt templates, retrieval parameters, language model selection — are managed under **Bot Settings → RAGForge**, available only for DocBrain bots. See [Bot Settings](/bot-settings) for the full configuration guide.

<Callout>
  DocBrain responses rely on indexed content. If a question has no matching content, DocBrain returns a fallback response. Keep your indexed material up-to-date for reliable answers.
</Callout>
