# Données en direct

Live Data affiche les conversations au moment où elles se déroulent. C'est la façon la plus rapide de repérer les problèmes après une modification fraîchement déployée, ou de surveiller en temps réel un lancement stratégique.

## Ce qui est inclus

Live Data fait remonter chaque conversation actuellement active sur l'ensemble de vos intégrations — chatbot web, appels vocaux, WhatsApp et tout canal connecté.

Pour chaque session, vous pouvez voir :

- Source et canal
- Heure de début
- Message actuel de l'utilisateur
- Réponse actuelle du bot
- Si l'utilisateur a demandé un agent

## Accès par rôle

La page Live Data est accessible aux rôles suivants :

- Team Owner
- Manager
- AIDA Admin
- AIDA User
- AIDA Dashboard

Le rôle AIDA Dashboard a un accès en lecture seule, utile pour les opérations de support ou les équipes Customer Success qui ont besoin de visibilité sans droits d'édition.

## Utiliser Live Data lors des déploiements

Une bonne pratique après avoir déployé une modification :

1. Ouvrez Live Data dans un onglet du navigateur.
2. Filtrez sur le canal où votre modification est active (par exemple, Webpage).
3. Surveillez les motifs inattendus — pics soudains de repli, conversations plus longues que d'habitude, plus de transferts vers des agents.
4. Si quelque chose semble anormal, consultez la [page d'entraînement](/intents-entities/train-conversations) pour les sessions spécifiques.

Pour une analyse historique ou agrégée, utilisez plutôt [Rapports et insights](/analytics/reports-insights).
