# Konversations-Knoten

Konversations-Knoten (CONV) steuern das sichtbare Hin und Her der Konversation – sie fragen Nutzer ab, rendern reichhaltige Cards und verzweigen basierend auf Nutzerantworten.

## Ask User

Fragt den Nutzer nach freier Texteingabe und speichert die Antwort in einer Variable.

**Felder:**
- **Message** – der Prompt, den der Nutzer sieht
- **Conditions** – Validierungsregeln für die Antwort
- **Fallback Message** – angezeigt, wenn die Validierung fehlschlägt
- **Variable** – wo die Antwort gespeichert wird
- **Suggestions** – optionale Quick-Reply-Chips
- **Show text area** – Vollständige Textarea statt einzeiliges Eingabefeld

## Gallery Card

Ein horizontal wischbarer Karten-Carousel – gut geeignet für Produktkataloge, Optionslisten oder Artikelvorschläge.

**Pro Card:**
- Card Image URL
- Float Image URL
- Title
- Subtitle
- Description
- Add Button (pro Card, mehrere möglich)

**Rund um den Carousel:**
- Message Before Card
- Message After Card
- Suggestions
- Fallback Message
- Variable
- Show text area

## Form Card

Ein strukturiertes Formular inline im Chat – erlaubt Nutzern, mehrere Felder gleichzeitig mit Validierung einzureichen.

**Feldtypen:**

| Feld | Verwendung |
|---|---|
| Header | Visuelle Gruppierung |
| TextField | Einzeiliger Text |
| Email | E-Mail-Adressen (automatisch validiert) |
| Text Area | Mehrzeiliger Text |
| Date Picker | Datumsangaben |
| Time Picker | Uhrzeiten |
| Numeric | Zahlen |
| Dropdown select | Einzelauswahl aus einer Liste |
| Checkbox group | Mehrfachauswahl |
| Radio button | Einzelauswahl, alle Optionen sichtbar |

**Action Buttons** unterstützen pro Button **Label**, **Validation text** und **Help**-Text. Die umgebende Konfiguration umfasst Message before/after card, Mandatory message und Show text area.

## Picture Card

Zeigt ein oder mehrere Bilder mit optionalem umgebenden Text.

**Felder:**
- Message Before Card
- Picture (Image URL)
- Add card – weitere Bilder anhängen
- Message After Card
- Suggestions
- Fallback Message
- Variable
- Show text area

## Video Card

Bettet ein Video per URL ein – ideal für Walkthroughs, Demos oder Onboarding-Inhalte.

**Felder:**
- Message Before Card
- Video Cards (Card Video URL pro Card hinzufügen)
- Message After Card
- Suggestions
- Fallback Message
- Variable
- Show text area

## Suggestion

Ein leichtgewichtiger Knoten, der Quick-Reply-Chips anzeigt. Nützlich für kurze Menüs oder Follow-up-Prompts.

**Felder:**
- Message
- Suggestions
- Fallback Message
- Variable
- Show text area

## Speak to an Agent

Übergibt die Konversation per Suggestion-Chip an einen Live-Agenten. Der angezeigte Text passt sich dem Verfügbarkeitsstatus des Live-Agenten an.

**Felder:**
- Suggestions
- Fallback Message
- Show text area

## LLM Classifier

Nutzt ein LLM, um die Antwort des Nutzers einem von mehreren Topics zuzuordnen und den Flow entsprechend zu verzweigen.

**Felder:**
- **Question** – was den Nutzer zu fragen ist
- **Description** – optionaler Kontext
- **Topic(s)** – jedes Topic definiert einen Ausgabebranch
- **Topic Description** – hilft dem LLM, das richtige Topic zu wählen
- **Fallback Message** – angezeigt, wenn kein Topic passt
- **Enable Fallback flow** – fügt einen Ausgang **LLM Fallback Flow** mit einem **Maximum Retry Limit** hinzu (1–5, Standard 3)
- **Variable** – speichert, welches Topic gewählt wurde
- **Topic(s) Variables** – topic-spezifischer Variablen-Storage
- **Show text area**

<Callout type="warning">
  Ohne aktivierten Fallback-Flow bricht der Flow nach **4 Retries** mit einer Standard-AIDA-Nachricht ab. Aktiviere den Fallback-Flow für jeden Classifier, bei dem du eine elegante Erholung von Fehlklassifikationen möchtest.
</Callout>
